# KASM

KASM is a small C11 assembler for Linux x86-64 programs. It uses an Intel-like
syntax and focuses on transparent output: direct ELF64 executables, ELF64 object
files, raw binary blobs, readable diagnostics, and tools that show what bytes and
ELF structures were produced.

KASM is not intended to replace NASM, GAS, FASM, or a full system toolchain. It is
best suited for small no-libc Linux programs, assembly experiments, binary layout
work, teaching, and projects where seeing the generated bytes matters.

## What KASM Can Do

- Assemble Linux x86-64 source files with a compact Intel-like syntax.
- Write direct static ELF64 executables without invoking `ld`.
- Write ELF64 relocatable object files for `ld`, `gcc -no-pie`, or `kasm link`.
- Write raw binary output for file-format headers and layout experiments.
- Assemble multiple `.asm` files independently into matching `.o` files.
- Link simple KASM-generated ELF64 objects with the built-in linker.
- Build small projects from a strict `kasm.toml` file.
- Expand includes, defines, macros, macro-local labels, and structs.
- Provide Linux syscall sugar and small include-based stdlib helpers.
- Generate explain output, maps, listings, symbol dumps, section dumps, and
  relocation dumps.
- Inspect KASM-generated ELF64 files and lightly disassemble supported encodings.
- Offer optional teaching, hint, and tiny-size modes.

## Supported Platform

KASM emits Linux x86-64 code. The assembler itself is portable C11 and can be
built on Linux, macOS, Windows with WSL, or Windows with a C toolchain, but the
programs it produces are Linux x86-64 ELF files.

## Build On Linux

Requirements:

- `make`
- a C11 compiler such as `gcc` or `clang`
- optional: `ld`, `readelf`, `objdump`, `nasm`, and `gcc` for comparison tests

```sh
git clone https://github.com/akluth/kasm.git
cd kasm
make
./kasm --version
make test
```

Install to `/usr/local`:

```sh
sudo make install
```

Install to a custom prefix:

```sh
make install PREFIX="$HOME/.local"
```

## Build On macOS

KASM can be compiled on macOS, but generated executables are Linux ELF64 files and
will not run natively on macOS.

Requirements:

- Xcode Command Line Tools or Homebrew LLVM
- `make`

```sh
xcode-select --install
git clone https://github.com/akluth/kasm.git
cd kasm
make CC=clang
./kasm --version
```

To run generated binaries, use a Linux VM, Docker container, or remote Linux
machine:

```sh
./kasm examples/01_hello_raw_syscall.asm -o hello
file hello
```

## Build On Windows

The recommended Windows setup is WSL because KASM targets Linux ELF64 output.

```powershell
wsl --install
wsl
```

Inside WSL:

```sh
sudo apt update
sudo apt install build-essential git
git clone https://github.com/akluth/kasm.git
cd kasm
make
make test
./kasm examples/01_hello_raw_syscall.asm -o hello
./hello
```

Native Windows builds are possible with a C11 compiler and `make`, but generated
programs still target Linux and should be run in WSL or another Linux
environment.

## Quick Start

Assemble and run the raw Linux syscall hello-world example:

```sh
make
./kasm examples/01_hello_raw_syscall.asm -o hello
./hello
```

Minimal source:

```asm
entry _start

section .text
_start:
    mov rax, 1
    mov rdi, 1
    lea rsi, [rel msg]
    mov rdx, msg_len
    syscall

    mov rax, 60
    xor rdi, rdi
    syscall

section .rodata
msg:
    db "Hello from KASM", 10
msg_len = $ - msg
```

## Command Line

```sh
kasm [options] input.asm
kasm main.asm util.asm syscalls.asm
kasm build [--config FILE] [--verbose] [--no-link]
kasm link file.o... -o app [--entry SYMBOL]
kasm inspect [--all] [--headers] [--segments] [--sections] [--symbols] [--relocs] file
kasm disasm [--section NAME] [--start ADDR] [--length N] file
```

Common options:

- `-o FILE`: write a named output file.
- `-f elf64|elf64-obj|obj|bin`: select the output format.
- `-I PATH`: add an include search path.
- `--tiny`, `-Oz`: prefer smaller safe encodings.
- `--no-tiny`: turn tiny mode off after an earlier tiny option.
- `--tiny-report`: print tiny-mode statistics.
- `--explain[=normal|verbose|deluxe]`: explain generated encodings.
- `--map FILE`: write a symbol/section map.
- `--list FILE`: write a listing with offsets and bytes.
- `--dump-symbols`, `--dump-sections`, `--dump-relocs`, `--dump-all`: print
  assembly-unit diagnostics.
- `--teach`, `--teach-level beginner|intermediate|deep`: print learning notes.
- `--hints`, `--hints=perf,abi,size`: print optional performance and ABI notes.
- `--no-syscall-sugar`: require raw syscall register setup.
- `--print-include-paths`, `--print-std-path`: show include roots.
- `--help`, `--version`: print CLI help or version information.

## Output Formats

Direct ELF64 executable, the default:

```sh
./kasm examples/01_hello_raw_syscall.asm -o hello
./hello
```

Relocatable object file:

```sh
./kasm examples/07_object_ld.asm -f obj -o object.o
ld object.o -o object
./object
```

Raw binary:

```sh
./kasm examples/09_struct_binary_header.asm -f bin -o header.bin
hexdump -C header.bin
```

Multiple input files:

```sh
./kasm main.asm util.asm syscalls.asm
ld main.o util.o syscalls.o -o app
```

Each input is assembled independently. `main.asm` becomes `main.o`,
`util.asm` becomes `util.o`, and so on. Cross-file linking should be done with
`ld` or `kasm link`.

## Project Builds

`kasm build` reads a strict `kasm.toml` file:

```toml
[project]
name = "hello"
type = "executable"

[build]
sources = ["src/main.asm", "src/util.asm"]
out_dir = "build"
output = "hello"
linker = "ld"
entry = "_start"
```

```sh
cd examples/project_hello
../../kasm build --verbose
./build/project_hello
```

Use `--no-link` to stop after object generation:

```sh
../../kasm build --no-link
```

Use the built-in linker for simple KASM-generated object files:

```sh
../../kasm build --internal-linker
../../kasm build --linker internal
```

## Includes, Macros, And Structs

KASM supports:

- `include "file.inc"`
- `define NAME value`
- simple macros with parameters
- macro-local labels
- `struc`, `sizeof`, and `offsetof`
- `db`, `dw`, `dd`, `dq`, `align`, and string data

Example:

```asm
struc Header
    .magic: db "KASM"
    .version: dd 1
    .payload_len: dq 0
endstruc

section .data
file_header:
    istruc Header
        at Header.version, dd 1
        at Header.payload_len, dq payload_end - payload
    iend
payload:
    db "data"
payload_end:
```

## Linux Syscall Sugar

Named syscall pseudo-instructions expand to the Linux x86-64 syscall ABI:

```asm
syscall write, STDOUT, msg, msg_len
syscall exit, 0
```

You can disable this layer with `--no-syscall-sugar` and write raw register setup
instead.

## Inline Stdlib Helpers

The bundled helpers under `lib/kasm` are include files, not libc and not a
runtime. They expand to ordinary assembly and Linux syscalls.

```asm
include "std/linux/io.asm"
include "std/linux/process.asm"

global _start
section .text
_start:
    kprintln "hello from kasm std"
    kexit 0
```

Show include roots:

```sh
./kasm --print-include-paths
./kasm --print-std-path
```

## Explain, Maps, And Listings

Explain mode shows how KASM encoded supported instructions:

```sh
./kasm examples/10_explain_demo.asm -o demo --explain=deluxe
```

Map and listing files:

```sh
./kasm examples/10_explain_demo.asm -o demo --map demo.map --list demo.lst
```

Diagnostic dumps:

```sh
./kasm examples/object_start.asm -f obj --dump-all -o object_start.o
```

## Binary Inspection And Disassembly

Inspect a generated object or executable:

```sh
./kasm inspect object_start.o
./kasm inspect --headers object_start.o
./kasm inspect --sections object_start.o
./kasm inspect --symbols object_start.o
./kasm inspect --relocs object_start.o
./kasm inspect --segments hello
./kasm inspect --all hello
```

Lightly disassemble supported encodings:

```sh
./kasm disasm hello
./kasm disasm --section .text hello
./kasm disasm --start 0x401000 --length 64 hello
```

The disassembler is intentionally small. Unknown bytes are printed as `db 0xNN`.

## Tiny Mode

Tiny mode chooses smaller encodings where doing so preserves semantics:

```sh
./kasm --tiny --tiny-report examples/tiny_exit.asm -o tiny_exit
```

It can shorten jumps, choose sign-extended imm8 forms, use accumulator immediate
forms, use `push imm8`, reduce memory displacements, and compact direct ELF
layout. It does not perform semantic rewrites such as changing `mov reg, 0` into
`xor reg, reg`.

## Teaching And Hints

Teaching mode prints a guided walkthrough:

```sh
./kasm --teach examples/teach_hello.asm -o teach_hello
./kasm --teach-level deep examples/teach_hello.asm -o teach_hello
```

Hints print optional notes without changing output:

```sh
./kasm examples/hints.asm -o hints --hints
./kasm examples/hints.asm -o hints --hints=perf,abi,size
```

## Current Limits

KASM is Linux x86-64 only. It has a limited instruction subset, limited
relocation types, no dynamic linker generation, no libc integration, no DWARF
debug info, and no full C preprocessor. Its inspection and disassembly tools are
for KASM-generated ELF64 files and are not full replacements for `readelf` or
`objdump`.

## Exit Codes

- `0`: success
- `1`: assembly or source error
- `2`: command-line usage error
- `3`: file or system error, reserved for future finer-grained handling
