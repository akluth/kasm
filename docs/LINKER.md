# Internal ELF64 Linker

KASM v2.0 includes a minimal built-in linker for KASM-generated ELF64 relocatable object files.

```sh
./kasm main.asm util.asm
./kasm link main.o util.o -o app
./app
```

The linker is intentionally small. It is meant for tiny no-libc Linux x86-64 programs, not as a replacement for GNU `ld` or `lld`.

## CLI

```sh
kasm link main.o util.o -o app
kasm link main.o util.o -o app --entry _start
```

Project mode can opt in:

```sh
kasm build --internal-linker
kasm build --linker internal
```

External linker mode remains supported and is still the default for `kasm build`.

## Supported Input

The internal linker supports ELF64 relocatable object files generated by KASM.

Supported sections:

- `.text`
- `.rodata`
- `.data`
- `.bss` as zero-filled data when present

Supported relocation types:

- `R_X86_64_64`
- `R_X86_64_PC32`
- `R_X86_64_32`
- `R_X86_64_32S`

KASM currently emits `R_X86_64_64` and `R_X86_64_PC32` for its supported object-mode constructs.

## Diagnostics

The linker reports the object file involved in each error where possible:

- malformed input object
- duplicate global symbol
- undefined symbol
- unsupported relocation target
- unsupported relocation type
- relocation overflow

## Layout

The output is a static Linux x86-64 ELF64 executable with deterministic page-aligned LOAD segments for text, read-only data, and writable data.

## Limitations

The internal linker does not implement dynamic linking, archive libraries, linker scripts, COMDAT, weak-symbol semantics, full section merging rules, or debug metadata. Use `ld` or `lld` for general-purpose linking.
