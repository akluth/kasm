# Binary Inspection

KASM includes small binary-learning tools for ELF64 files generated by KASM.
They are intentionally scoped: useful for KASM objects and simple Linux
executables, not a full `readelf` or `objdump` replacement.

## Inspect

```sh
./kasm inspect file.o
./kasm inspect --headers file.o
./kasm inspect --sections file.o
./kasm inspect --symbols file.o
./kasm inspect --relocs file.o
./kasm inspect --segments hello
./kasm inspect --all hello
```

`inspect` validates ELF magic, ELF64 class, little-endian encoding, and table
bounds before reading. It prints:

- ELF header: class, endianness, type, machine, entry, program header offset,
  section header offset
- program headers: type, flags, virtual address, file offset, file size, memory
  size, alignment
- section headers: name, type, flags, address, file offset, size, alignment
- symbols: name, value, size, binding, type, section
- relocations: section, offset, type, symbol, addend

## Disasm

```sh
./kasm disasm hello
./kasm disasm --section .text hello
./kasm disasm --start 0x401000 --length 64 hello
```

The disassembler-light decodes common instructions KASM can emit:

- `mov`, `lea`
- `push`, `pop`
- `call`, `ret`
- `jmp`, common `jcc`
- `cmp`, `test`
- `add`, `sub`, `xor`, selected group-immediate forms
- `syscall`, `nop`

Unknown bytes are emitted as `db 0xNN`. Malformed or truncated inputs fail with
a diagnostic instead of reading past the file buffer.

## Encoding Exploration

Use normal assembly with explain mode for a precise byte breakdown:

```sh
cat > /tmp/one.asm <<'ASM'
entry _start
section .text
_start:
    mov rax, 1
ASM
./kasm /tmp/one.asm --explain=deluxe -f bin -o /tmp/one.bin
```

This prints bytes, prefixes, opcode, ModRM/SIB where present, displacement,
immediate, relocation notes, and the selected encoding form.

## Example Flow

```sh
./kasm examples/object_start.asm -f obj -o object_start.o
./kasm inspect --all object_start.o
ld object_start.o -o object_start
./kasm inspect --segments object_start
./kasm disasm object_start
```
