# ELF Surgeon Mode

KASM includes `--elf-info`, a small built-in ELF anatomy report for generated ELF64 executables and relocatable objects.

```sh
./kasm --elf-info examples/hello.asm -o hello
./kasm --elf-info -f obj examples/object_start.asm -o object_start.o
```

The mode prints readable text and does not change generated bytes. It reports:

- ELF header summary
- program headers for direct executables
- generated sections with offsets, addresses, sizes, flags, and alignment
- symbols known to KASM
- relocations for object output
- an entry point explanation

For direct executables, the entry explanation names the resolved entry label and the address where the Linux loader starts execution.

For object files, KASM reports that no loader entry point exists yet because `ld` or another linker chooses the final entry when objects are linked.

`--elf-info` is intentionally an educational view over KASM's generated layout. Use `readelf` or `objdump` when you need a complete external ELF parser.
