# Your First Tiny Linux ELF With KASM

KASM includes Teaching Mode:

```sh
./kasm --teach examples/teach_hello.asm -o teach_hello
```

Teaching Mode prints a guided walkthrough after assembly. It keeps the output bytes unchanged and explains:

- the program entry label
- sections such as `.text` and `.rodata`
- each instruction or data directive
- generated bytes in the default and deep levels
- label references
- Linux syscall ABI registers
- the ELF output type and entry point
- how to run the generated executable

Levels:

```sh
./kasm --teach-level beginner examples/teach_hello.asm -o teach_hello
./kasm --teach-level intermediate examples/teach_hello.asm -o teach_hello
./kasm --teach-level deep examples/teach_hello.asm -o teach_hello
```

`beginner` focuses on concepts and leaves most encoding bytes out.

`intermediate` is the default and includes generated bytes.

`deep` adds source-line/offset traces and embeds the ELF Surgeon summary.

The mode is intentionally concise: when KASM cannot yet explain something, it says `not yet explained` instead of pretending.
